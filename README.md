# EG-GenRM: Entropy-Guided Generative Reward Model for Mitigating Reasoning Hallucinations

> **STAT GR5293 (Sec 007) — GenAI Course Project, Columbia University, Spring 2026**
> Team: Ruimin Zhang (`rz2737`), Menglei Zhang (`mz3129`), Hantian Zhang (`hz3101`)

EG-GenRM augments a generative verifier (GenRM) with **entropy-masked supervision** so that fine-tuning gradient flow is restricted to the top-20 % highest-entropy "forking" tokens of each completion. The verifier is trained on top of `Qwen/Qwen2.5-1.5B-Instruct` via 4-bit QLoRA and combined with a **conflict-guarded majority-vote selection rule** at inference time. On a same-split held-out test of 38 questions, EG-GenRM reaches **100.0 %** final-answer-selection accuracy (Oracle = 100.0 %, Majority Voting = 94.7 %, Standard-QLoRA verifier = 97.4 %).

## Highlights

- **Novel loss**: `EntropyMaskedLoss` — gradients flow only through the top-20 % highest-entropy completion tokens, computed *online* from the verifier's own logits at every training step.
- **Same-split fair comparison** of 7 strategies (Random / Majority Voting / Teacher-GenRM / Teacher-filtered-Majority / Method A / Method B / Oracle) on a single 38-question test set with **no question leakage** across train / val / test.
- **End-to-end pipeline**: candidate generation → teacher annotation → SFT data construction → QLoRA fine-tuning → inference-time selection → bootstrap statistical analysis → error-transition analysis.
- **Resource-efficient**: full Method B training takes < 16 min on a single A100-40GB; only **9.2 M / 1.55 B (0.59 %)** of parameters are trainable.
- **Reproducible**: every figure and table in the final report is regenerated by re-running the notebooks end-to-end. Results CSVs are committed under `results/`.

The 186 complete questions are then split **group-aware 70/15/15** into:

| Split | Questions | Candidates |
|-------|-----------|------------|
| train | 113       | 451        |
| val   | 38        | 148        |
| test  | 38        | 152        |

> Group-aware splits are enforced via `sklearn.model_selection.GroupShuffleSplit` keyed on `question_norm`, with explicit `assert` checks for cross-split leakage.

## Method Summary

### Method A — Standard QLoRA Verifier (Ablation Baseline)

| Hyperparameter | Value |
|----------------|-------|
| Base model | `Qwen/Qwen2.5-1.5B-Instruct` |
| Quantization | 4-bit NF4 (BitsAndBytes), double-quant, fp16 compute |
| LoRA target modules | `q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj` |
| LoRA r / α / dropout | 8 / 16 / 0.05 |
| Trainable params | 9 232 384 / 1 552 946 688 (0.59 %) |
| Epochs | 3 |
| Learning rate | 2e-4 (warmup ratio 0.05) |
| Effective batch size | 1 × 8 (grad accum) |
| Max sequence length | 1024 |
| Loss | Standard token-level cross-entropy on completion tokens (prompt tokens masked with `-100`) |

The output format is restricted to a **single line** — `<final_verdict>Yes</final_verdict>` or `<final_verdict>No</final_verdict>` — to avoid format drift.

### Method B — EG-GenRM (Core Contribution)

Identical base / LoRA / data / split / hyper-parameters as Method A. **Only the loss function differs.**

Predictive entropy is computed online from the verifier's own logits at every training step:

```
H_t        = -Σ_v softmax(logits)_v · log_softmax(logits)_v   # per-token Shannon entropy
threshold  = quantile(H_t over completion tokens, 1 - 0.20)   # top-20% cutoff
M_t        = (H_t ≥ threshold) AND (label_t ≠ -100)           # binary mask
L_EG-GenRM = - Σ_t M_t · log P(y_t | x, y_<t)  /  Σ_t M_t     # mask-normalized CE
```

Implemented as `EntropyMaskedTrainer(transformers.Trainer)` overriding `compute_loss()` (see `src/train/train_method_b.py` and Part 9 of the notebook).

### Inference-time selection rules

| Rule | Description |
|------|-------------|
| **Naive** | Pick a random candidate from the verifier's *Yes* pool; fall back to random if empty. |
| **Filtered Majority** | Within the *Yes* pool, take the most-common candidate answer; tie-break by global support across all 4 candidates, then by random. |
| **Conflict-Guarded Majority (Method B only)** | If the *Yes*-pool majority conflicts with the global majority (and the global majority is ≥ 2 votes and supported by at least one *Yes* candidate), defer to the global majority. Mitigates the +8 FP regression introduced by Method B's higher recall. |

## Key Results

### Candidate-level verifier accuracy (test cands, n = 152)

| Method | Acc | Precision | Recall | F1 |
|--------|-----|-----------|--------|----|
| Method A — Standard QLoRA | 72.4 % | 66.7 % | 94.7 % | 0.783 |
| **Method B — EG-GenRM** | **73.0 %** | 66.4 % | **98.7 %** | **0.794** |

### Final-answer-selection accuracy (same-split held-out, n = 38)

| Strategy | Accuracy |
|----------|---------:|
| Random | 50.2 % |
| Majority Voting | 94.7 % |
| Teacher GenRM (naive) | 92.1 % |
| Teacher-filtered Majority | 100.0 % |
| Method A filtered majority | 97.4 % |
| **Method B EG-GenRM guarded majority** | **100.0 %** |
| Best-of-N Oracle (upper bound) | 100.0 % |

### Statistical analysis (paired bootstrap, 2 000 iterations)

- Method B − Method A = **+2.6 pp** (95 % CI **[0.0 %, 7.9 %]**, one-sided p = 0.370)
- Reported transparently as a **positive but statistically inconclusive** trend at n = 38.

### Efficiency

| Strategy | Verifier calls | Calls / question |
|----------|---------------:|-----------------:|
| Majority Voting | 0 | 0.0 |
| Teacher GenRM (all candidates) | 744 | 4.0 |
| **Method B EG-GenRM (test only)** | **152** | **0.8** |
